# Contributing to Company Portal

Thank you for your interest in contributing! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/obeliskportal.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Commit: `git commit -m 'Add your feature'`
6. Push: `git push origin feature/your-feature-name`
7. Open a Pull Request

## Development Setup

See the [README.md](README.md) for detailed setup instructions.

## Code Style

- Use TypeScript for all new code
- Follow existing code patterns
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused

## Commit Messages

Use clear, descriptive commit messages:
- `feat: Add employee KTP upload`
- `fix: Resolve payroll calculation bug`
- `docs: Update deployment guide`
- `refactor: Simplify report generation`

## Pull Request Process

1. Ensure your code follows the project's style
2. Update documentation if needed
3. Test your changes thoroughly
4. Ensure all tests pass (if applicable)
5. Request review from maintainers

## Security

**NEVER** commit:
- `.env` files
- API keys or secrets
- Database credentials
- Personal information
- Any sensitive data

If you accidentally commit sensitive data, contact the maintainers immediately.

## Questions?

Open an issue for questions or discussions about features.
